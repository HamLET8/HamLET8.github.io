Day 12 · SSD / FTL 代码设计判断

SLC Cache / WriteBooster 的边界：写入加速上下文不应污染主 FTL 写路径

今天的核心判断是：WriteBooster 是写入加速上下文，不应让主 FTL 写路径理解 SLC/TLC 迁移细节。 主路径只表达写入意图和一致性要求；SLC 分配、cache entry、destage、flush、回收和恢复都应收敛在独立边界内。

一句话判断 WriteBooster 管加速语义，主 FTL 管逻辑一致性 涉及：SLC Cache / Host Write / Mapping / Journal / GC / Recovery 关键词：bounded context · write route · cache entry · destage · flush protocol · QoS

今日设计判断

主 FTL 写路径不应该出现 if (slc_mode) 分支来理解 SLC/TLC 迁移。 它应该向 write router 提交写入请求，由 WriteBooster 决定 cache、fallback、backpressure 或 bypass。

今日涉及模块

SLC Cache manager、WriteBooster、Host Write、Mapping Table、Metadata Journal、 FTL Recovery、GC、Free Block Pool、QoS Scheduler、NAND Program / Erase Flow。

今日阅读重点：加速路径也是一个领域边界

WriteBooster 往往从一个性能优化开始，但很快会触及一致性、读后写、掉电恢复、缓存清空、TLC destage、 QoS 预算和空间水位。如果这些知识进入主写路径，主路径会同时承担逻辑写入和加速策略，最终变得不可维护。

思想来源 今天必须掌握的概念 落到 SSD / FTL 的判断
《软件设计哲学》 信息隐藏要求把易变策略藏在深模块内。SLC/TLC 细节是加速实现，不是主写接口。 FTL Core 只依赖 write_route_submit()，不直接理解 cache block、destage cursor、SLC free pool。
《程序员修炼之道》 正交性和可撤退性要求优化路径能被禁用、降级或替换，而不改变主路径语义。 WriteBooster full 时返回 bypass/backpressure 决策；主写路径仍能走普通 TLC 写入。
《设计的设计》 原型会演进成系统的一部分。临时性能优化如果没有边界，会成为永久复杂度。 把 flush protocol、destage policy 和 recovery record 设计成明确协议，而不是在主写函数里堆分支。
《领域驱动设计》 上下文映射要求不同子域使用各自语言，并通过稳定契约协作。 WriteBooster 使用 cache entry、destage job、SLC segment；Mapping 使用 LBA/PPA/current seq；二者通过 mapping delta 协作。

概念融合解释：WriteBooster 是上下文，不是 if 分支

四本书在这里形成的统一判断是：性能优化不能破坏概念边界。SLC Cache 可以改变物理写入位置和写入时机， 但不能改变 FTL 对外的一致性语义。主写路径只关心“这次写入是否提交、读路径能否读到最新版本、掉电后能否恢复”； WriteBooster 关心“是否写入 SLC、何时 destage 到 TLC、如何节流、如何回收 SLC 空间”。

统一判断： WriteBooster 的输出应该是领域契约：cache commit、destage delta、flush result、bypass decision。 它不能把 SLC/TLC 状态字段泄漏给 FTL Core；FTL Core 也不能绕过 WriteBooster 直接操作 cache metadata。

SSD / FTL 设计场景

Host Write 进入 SLC

Write Router 根据空间水位、QoS 和写入大小决定进入 SLC。主 FTL 只接收提交结果，不理解 SLC open segment。

Read After Write

读路径通过 Mapping 或 overlay 查询最新版本；如果 LBA 最新版本在 SLC，返回 cache PPA，而不是让 Host Read 扫描 cache。

Destage 到 TLC

后台 destage 构造从 SLC PPA 到 TLC PPA 的 mapping delta，先 journal，再条件提交，再释放 SLC entry。

SLC Full / Backpressure

SLC 空间不足时，WriteBooster 返回 bypass 或 backpressure 决策；主路径不直接调用 SLC GC。

Power Loss Recovery

Recovery replay cache commit 和 destage record，识别已提交 SLC entry、未完成 destage 和可回收 orphan cache page。

GC 协作

TLC GC 不选择 SLC cache block；SLC 回收由 WriteBooster 的 flush / invalidate 协议驱动。

坏味道： 这段代码把 SLC 分配、TLC fallback、mapping 更新、cache full 处理、destage 触发全部塞进主写路径。 主路径开始理解缓存实现，WriteBooster 不再是独立边界。
反例：ftl_write 充满 SLC/TLC 分支 C 
int ftl_write_bad(struct ftl_dev *ftl, u64 lba, const void *buf, u32 len)
{
	struct ftl_ppa ppa;
	struct ftl_ppa old_ppa;
	int ret;

	if (ftl->slc_enabled && len <= ftl->slc_max_write) {
		if (ftl->slc_free_pages == 0) {
			ret = slc_flush_some_pages(ftl);
			if (ret)
				goto write_tlc;
		}

		ppa = ftl->slc_open_block->next_ppa;
		ret = nand_program_slc(ftl->nand, &ppa, buf, len);
		if (ret)
			return ret;

		old_ppa = ftl->l2p[lba];
		ftl->l2p[lba] = ppa;
		ftl->slc_map[lba] = ppa;
		ftl->slc_free_pages--;
		ftl->slc_dirty_pages++;

		if (ftl->slc_dirty_pages > ftl->slc_flush_threshold)
			schedule_destage_thread(ftl);

		block_invalidate_page(ftl->bm, &old_ppa);
		return 0;
	}

write_tlc:
	ret = block_alloc_tlc_page(ftl->bm, &ppa);
	if (ret)
		return ret;

	ret = nand_program_tlc(ftl->nand, &ppa, buf, len);
	if (ret)
		return ret;

	old_ppa = ftl->l2p[lba];
	ftl->l2p[lba] = ppa;
	block_invalidate_page(ftl->bm, &old_ppa);
	return 0;
}
设计问题 代码表现 后果
上下文泄漏 主路径直接读写 slc_open_block、slc_map、dirty count。 SLC 实现一变，主写路径和恢复路径都要改。
Mapping 所有权破坏 直接替换 l2p。 无法做 journal、条件提交、destage replay 和 read-after-write 统一解析。
错误路径不可恢复 program 后没有 cache commit record。 掉电后不知道 SLC page 是已提交、未提交还是 orphan。
QoS 和策略混杂 写路径直接触发 flush / destage。 前台延迟、后台 flush、空间水位互相污染，无法单独调度。

改进代码：WriteBooster 通过写入路由和协议协作

改进版把 WriteBooster 作为独立上下文。FTL Core 提交写入请求，Write Router 选择 cache、bypass 或 backpressure。 Mapping、Journal、Block Manager 仍通过清晰接口维护一致性。

write_booster.h：上下文边界接口 C 
#ifndef FTL_WRITE_BOOSTER_H
#define FTL_WRITE_BOOSTER_H

#include <linux/types.h>
#include <linux/errno.h>

enum wb_route {
	WB_ROUTE_CACHE = 0,
	WB_ROUTE_BYPASS,
	WB_ROUTE_BACKPRESSURE,
};

enum wb_entry_state {
	WB_ENTRY_RESERVED = 0,
	WB_ENTRY_PROGRAMMED,
	WB_ENTRY_COMMITTED,
	WB_ENTRY_DESTAGING,
	WB_ENTRY_RELEASED,
};

struct wb_write_req {
	u64 lba;
	const void *buf;
	u32 len;
	u32 qos_class;
	bool force_fua;
};

struct wb_write_result {
	enum wb_route route;
	struct ftl_ppa visible_ppa;
	u64 mapping_seq;
	bool recovery_required;
};

struct write_booster;

int wb_submit_write(struct write_booster *wb, const struct wb_write_req *req,
		    struct wb_write_result *result);
int wb_schedule_destage(struct write_booster *wb, u32 max_entries);
int wb_replay_cache_record(struct write_booster *wb,
			   const struct journal_wb_record *record);

#endif
write_booster.c：cache commit 与 destage 分离 C 
int wb_submit_write(struct write_booster *wb, const struct wb_write_req *req,
		    struct wb_write_result *result)
{
	struct wb_cache_entry entry;
	struct journal_wb_record record;
	int ret;

	ret = wb_route_write(wb, req);
	if (ret == WB_ROUTE_BYPASS) {
		result->route = WB_ROUTE_BYPASS;
		return 0;
	}
	if (ret == WB_ROUTE_BACKPRESSURE)
		return -EAGAIN;
	if (ret < 0)
		return ret;

	ret = wb_reserve_cache_entry(wb, req, &entry);
	if (ret)
		return ret;

	ret = nand_program_slc(wb->nand, &entry.ppa, req->buf, req->len);
	if (ret)
		goto err_program;

	entry.state = WB_ENTRY_PROGRAMMED;
	wb_build_cache_commit_record(&entry, req, &record);

	ret = journal_append_wb_record(wb->journal, &record);
	if (ret)
		goto err_journal;

	ret = map_commit_cache_entry(wb->map, req->lba, &entry.ppa,
				     record.new_mapping_seq);
	if (ret)
		goto err_mapping;

	entry.state = WB_ENTRY_COMMITTED;
	wb_commit_entry(wb, &entry);

	result->route = WB_ROUTE_CACHE;
	result->visible_ppa = entry.ppa;
	result->mapping_seq = record.new_mapping_seq;
	return 0;

err_mapping:
	wb_mark_recovery_required(wb, WB_RECOVERY_CACHE_COMMIT);
	return ret;

err_journal:
	wb_mark_orphan_entry(wb, &entry);
	return ret;

err_program:
	wb_release_reserved_entry(wb, &entry);
	return ret;
}

int wb_destage_one(struct write_booster *wb, struct wb_cache_entry *entry)
{
	struct mapping_destage_delta delta;
	struct ftl_ppa tlc_ppa;
	int ret;

	ret = block_alloc_tlc_page(wb->bm, &tlc_ppa);
	if (ret)
		return ret;

	ret = nand_copy_slc_to_tlc(wb->nand, &entry->ppa, &tlc_ppa);
	if (ret)
		goto err_copy;

	map_build_destage_delta(wb->map, entry->lba, &entry->ppa,
				&tlc_ppa, &delta);

	ret = journal_append_destage_delta(wb->journal, &delta);
	if (ret)
		goto err_journal;

	ret = map_commit_destage_if_current(wb->map, &delta);
	if (ret)
		goto err_mapping;

	wb_release_committed_entry(wb, entry);
	return 0;

err_mapping:
	wb_mark_recovery_required(wb, WB_RECOVERY_DESTAGE_REPLAY);
	return ret;

err_journal:
	block_discard_uncommitted_page(wb->bm, &tlc_ppa);
	return ret;

err_copy:
	block_apply_program_failure(wb->bm, &tlc_ppa, ret);
	return ret;
}

协议表与思想落点表

WriteBooster 与主 FTL 边界表

协作点 主 FTL 负责 WriteBooster 负责 禁止做法
写入路由 提交 LBA、payload、QoS、FUA 等写入意图。 判断 cache、bypass、backpressure。 主路径读取 SLC free page 或 open segment。
可见性提交 依赖 Mapping 的当前事实。 构造 cache commit record 并请求 Mapping 条件提交。 直接修改 L2P 或维护第二份 slc_map。
读后写 按 Mapping/overlay 查询最新 PPA。 保证 committed cache entry 可被查询。 Host Read 扫描 SLC cache 内部链表。
Destage 不感知 SLC 到 TLC 的迁移细节。 后台 copy、journal、mapping commit、释放 cache entry。 主写路径同步执行大量 SLC flush。
恢复 挂载时调用 Recovery 统一接管。 提供 cache record replay 和 orphan entry 处理。 Recovery 猜测 SLC block 的内部 cursor。

WriteBooster 状态与错误矩阵

阶段 失败 领域解释 补偿动作 返回语义
Reserve cache entry SLC 空间不足。 加速资源不可用，但主写语义仍可继续。 返回 bypass 或 backpressure。 0 with bypass 或 -EAGAIN
Program SLC program fail。 cache entry 未提交，不应对 Mapping 可见。 释放 reserved entry，必要时隔离 SLC block。 -EIO
Journal cache commit 记录失败。 SLC page 已写但不可恢复。 标记 orphan cache entry，等待回收。 -EIO
Destage commit old SLC PPA 已非当前。 前台写已覆盖该 LBA，destage 过期。 释放旧 cache entry 或标记 stale。 -ESTALE 可收敛
Flush barrier FUA 或 shutdown flush 未完成。 主机要求持久化边界尚未满足。 阻塞、降级只读或返回错误。 -EIO / -EAGAIN

四本书思想落点表

书 思想 SSD / FTL 设计位置 它约束了什么 它帮助避免什么问题
《软件设计哲学》 隐藏易变实现，暴露稳定接口。 write_route_submit、wb_submit_write、destage API。 主路径不理解 SLC segment、dirty count、destage cursor。 避免加速实现泄漏导致变更放大。
《程序员修炼之道》 正交性、可撤退性和务实降级。 bypass/backpressure、flush protocol、recovery_required。 WriteBooster 可禁用或降级，主 FTL 语义不变。 避免性能优化成为不可关闭的风险源。
《设计的设计》 原型到产品要建立设计约束。 SLC cache entry 状态机、destage job、journal record。 临时优化必须有明确协议和恢复边界。 避免性能补丁长期污染主写路径。
《领域驱动设计》 上下文映射和领域语言分离。 WriteBooster context 与 Mapping / GC / Recovery 的协作契约。 SLC 语言只在加速上下文内使用，对外输出 mapping delta 和 flush result。 避免 SLC/TLC 细节在 FTL Core、GC、Recovery 中到处传播。
